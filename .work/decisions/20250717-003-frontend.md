# ADR-003: React + Vite frontend

**Status:** Decided · **Date:** 2025-07-17

## Context

The platform requires a rich document review UI with per-question editing, source citation display, approval workflows, and admin panels. The target audience includes bid managers and reviewers who expect a desktop-class web experience.

## Decision

Use **React with TypeScript** built on **Vite** for the frontend.

## Consequences

- **Positive:** Rich component ecosystem (especially for document-like UIs); TypeScript catches common bugs; Vite provides fast HMR during development; broad hiring pool.
- **Negative:** Bundle size can grow large without code-splitting (mitigated by route-based lazy loading); state management requires deliberate choice (React Context + hooks sufficient for V1).
- **Neutral:** Vite replaces Create React App (deprecated) and Next.js (heavier for a non-SEO app).

## Alternatives considered

| Alternative | Reason rejected |
|-------------|-----------------|
| Next.js | Heavier SSG/SSR not needed for authenticated SPA; more complexity for solo dev |
| Vue/Nuxt | Smaller component ecosystem for document-review UIs |
| Svelte/SvelteKit | Smaller hiring pool; fewer battle-tested UI library options |
